OCR 文字识别模块

基于 PaddleOCR 的高精度文字识别解决方案。无需训练，开箱即用， 支持 80+ 种语言，提供文本检测、方向分类、文字识别全流程，支持 TensorRT 高性能部署。

🏗️ 引擎：PaddleOCR
✨ 无需训练
🌍 80+ 语言
📅 更新：2025年1月
📋

功能概览

OCR（Optical Character Recognition，光学字符识别）模块是 DL-Hub 的核心组件之一， 提供从图像中提取文字的完整解决方案。基于百度 PaddleOCR，无需任何训练即可直接使用， 预训练模型已能很好地处理大多数场景。

✅
开箱即用：PaddleOCR 提供高质量预训练模型，覆盖中文、英文等 80+ 种语言， 无需收集数据、无需训练，安装后即可直接使用！

快速识别

快速识别是最常用的功能，用于对单张图片进行 OCR 识别。

使用方法

  1. 在左侧面板选择推理引擎（默认 PaddleOCR）
  2. 根据需要开启 GPU 加速和方向分类
  3. 上传图片到"输入图片"区域
  4. 调整检测阈值和识别阈值（可选）
  5. 点击"开始识别"按钮

输出结果

  • 可视化图像：原图上标注检测框和编号
  • 文本内容：识别出的所有文字，按检测顺序排列
  • 详细列表：每个文本区域的序号、内容、置信度
  • 统计信息：文本数量、总耗时、检测耗时、识别耗时

参数说明

参数 默认值 说明
检测阈值 0.3 文本区域检测阈值，越低检出越多
识别阈值 0.5 识别置信度过滤阈值，低于此值的结果会被过滤
方向分类 关闭 开启后可识别竖排、倾斜文字
绘制样式 轮廓线 检测框绘制方式：轮廓线/填充混合

批量处理

批量处理功能用于对文件夹中的多张图片进行 OCR 识别，自动保存结果。

使用流程

  1. 输入包含图片的文件夹路径
  2. 设置输出目录（默认 ./ocr_results）
  3. 选择导出格式（CSV / JSON）
  4. 点击"开始处理"，可随时查看进度

文本检测

文本检测仅定位图像中的文本区域，不进行识别。输出四边形边界框坐标， 适合需要自定义后处理或只关注文本位置的场景。

检测算法

PaddleOCR 使用 DBNet（Differentiable Binarization）作为检测模型， 具有以下特点：

  • 端到端可微分，训练更简单
  • 支持任意形状文本（弯曲、倾斜）
  • 检测速度快，精度高

检测参数

参数 默认值 说明
det_limit_side_len 960 检测时图像最长边限制
det_db_thresh 0.3 二值化阈值
det_db_box_thresh 0.6 文本框得分阈值
det_db_unclip_ratio 1.5 文本框扩展比例
📖

文本识别

文本识别用于对已裁剪好的文本图像进行识别，输出文字内容和置信度。

识别算法

PaddleOCR 使用 CRNN + CTC 架构进行文字识别：

  • CRNN：卷积循环神经网络，提取序列特征
  • CTC：连接时序分类，处理不定长序列

识别参数

参数 默认值 说明
rec_image_shape 3,48,320 识别输入尺寸 (C,H,W)
rec_batch_num 6 批处理大小
use_space_char True 是否识别空格

导出命令

Bash 
# 安装 paddle2onnx
pip install paddle2onnx

# 导出检测模型到 ONNX
paddle2onnx --model_dir ~/.paddleocr/whl/det/ch/ch_PP-OCRv4_det_infer \
            --model_filename inference.pdmodel \
            --params_filename inference.pdiparams \
            --save_file det_model.onnx \
            --opset_version 14

# 导出识别模型到 ONNX

            --model_filename inference.pdmodel \
            --params_filename inference.pdiparams \
            --save_file rec_model.onnx \
            --opset_version 14

TensorRT 部署

TensorRT 是 NVIDIA 的高性能深度学习推理优化器，可以显著提升 OCR 推理速度。

构建 TensorRT 引擎

Bash 
# 检测模型 (固定输入)
trtexec --onnx=det_model.onnx \
        --saveEngine=det_model.engine \
        --fp16 \
        --workspace=4096

# 识别模型 (动态宽度)
trtexec --onnx=rec_model.onnx \
        --saveEngine=rec_model.engine \
        --fp16 \
        --workspace=4096 \
        --minShapes=x:1x3x48x100 \
        --optShapes=x:1x3x48x320 \
        --maxShapes=x:1x3x48x800

部署所需文件

API 使用

基本使用

Python 
from paddleocr import PaddleOCR
import cv2

# 初始化 OCR
ocr = PaddleOCR(use_angle_cls=True, lang='ch')

# 识别图片
result = ocr.ocr('test.jpg', cls=True)

# 解析结果
for line in result[0]:
    box = line[0]      # 四边形坐标 [[x1,y1], [x2,y2], [x3,y3], [x4,y4]]
    text = line[1][0]  # 识别文本
    score = line[1][1] # 置信度
    print(f"文本: {text}, 置信度: {score:.2%}")

仅检测

Python 
# 仅执行文本检测
result = ocr.ocr('test.jpg', rec=False)

for box in result[0]:
    print(f"检测框: {box}")

仅识别

Python 
# 仅执行文本识别（输入已裁剪的文本图像）
result = ocr.ocr('cropped_text.jpg', det=False)

常见问题

需要自己训练模型吗？
不需要！PaddleOCR 提供的预训练模型已经能很好地处理大多数场景， 包括印刷体、手写体、各种复杂背景等。只有在特殊场景（如特殊字体、特殊排版）下才需要微调。
如何处理竖排文字？
启用方向分类器即可：PaddleOCR(use_angle_cls=True)。 方向分类器会自动检测文字方向并进行旋转校正。
识别精度不够怎么办？
1. 确保输入图像清晰度足够
2. 调低检测阈值以检出更多文本区域
3. 使用更高版本的模型（PP-OCRv4）
4. 对特定场景可以考虑微调模型
TensorRT 部署需要哪些文件？
需要三个文件：
• det_model.engine - 检测引擎
• rec_model.engine - 识别引擎
• ppocr_keys_v1.txt - 字符字典（约 6000+ 中英文字符）
支持哪些图像格式？
支持常见的图像格式：JPG、JPEG、PNG、BMP、TIFF 等。 建议使用 JPG 或 PNG 格式，分辨率不低于 720p 以获得最佳效果。
