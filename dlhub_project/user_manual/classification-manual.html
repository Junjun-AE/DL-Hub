基于 PyTorch 和 TIMM 库构建的专业图像分类训练平台，提供 15+ 精选预训练模型， 支持从数据准备、模型训练、模型转换到生产部署的完整工作流程。无论您是深度学习新手还是资深算法工程师， 都能快速上手并获得专业级的训练效果。

概述

图像分类模块是 DL-Hub 深度学习平台的核心组件之一，专为工业级图像分类任务设计。 该模块基于 PyTorch 深度学习框架和 TIMM（PyTorch Image Models）库构建， 集成了业界最先进的预训练模型和训练技术，让用户无需编写代码即可完成专业级的模型训练。

无论您是需要进行产品质检、医疗影像分析、农业病害识别，还是其他图像分类任务， 本模块都能提供从数据准备到模型部署的端到端解决方案。通过直观的 Web 界面， 您可以轻松配置训练参数、监控训练进度、转换模型格式，并进行批量推理测试。

快速入门

本节将指导您在 5-10 分钟内完成第一次模型训练。我们将使用一个简单的示例， 帮助您熟悉整个训练流程。在开始之前，请确保已按照「环境配置」章节完成环境安装。

1
准备数据集

将您的图片按类别整理到不同的文件夹中。文件夹的名称将作为类别名称，系统会自动识别。

dataset/ ├── cat/ # 类别1：猫 │ ├── cat_001.jpg │ ├── cat_002.jpg │ ├── cat_003.png │ └── ... # 更多图片 ├── dog/ # 类别2：狗 │ ├── dog_001.jpg │ ├── dog_002.jpg │ └── ... └── bird/ # 类别3：鸟 ├── bird_001.jpg └── ...
💡
数据集要求
每个类别建议至少 50 张图片。图片格式支持 JPG、PNG、BMP、WebP 等常见格式。 系统会自动将数据集按比例划分为训练集和验证集。
2
启动训练界面

您可以通过以下两种方式启动图像分类模块：

bash
# 方式一：通过 DL-Hub 启动（推荐）
# 在 DL-Hub 主界面点击「图像分类」任务卡片

# 方式二：独立运行
cd model_image_classification
python app.py

# 方式三：指定端口运行
python app.py --port 7861

# 启动后，在浏览器中访问显示的地址（通常是 http://127.0.0.1:7860）
3
配置训练参数

在 Web 界面的「训练」标签页中进行以下配置：

  1. 数据集路径：输入或选择您准备好的数据集文件夹路径
  2. 模型选择：选择模型系列（推荐 EfficientNet）和规模（推荐「中」）
  3. 训练参数：首次使用可保持默认值，或根据需要调整：
    • 训练轮数（Epochs）：建议 30-50 轮
    • 批次大小（Batch Size）：根据显存调整，通常 16-32
    • 学习率：默认 0.001，通常无需修改
  4. GPU选择：如有多张显卡，可选择指定 GPU
4
开始训练并监控

点击「开始训练」按钮后，您可以实时监控训练进度：

  • 训练日志：显示每个 epoch 的损失值、准确率、学习率等信息
  • 损失曲线：实时绘制训练损失和验证损失的变化趋势
  • 准确率曲线：展示验证集准确率的提升过程
  • 最佳模型：系统会自动保存验证准确率最高的模型

训练过程中可以随时点击「停止训练」按钮安全中断，系统会保存当前检查点。

5
使用训练好的模型

训练完成后，您可以：

  • 批量推理：切换到「推理」标签页，对新图片进行分类预测
  • 模型转换：切换到「转换」标签页，将模型转换为 ONNX/TensorRT 等部署格式
  • 导出模型：在输出目录中找到 best_model.pth 文件
✅
快速验证建议
首次使用时，建议选择较小的模型（如 MobileNetV3-Small）和较少的训练轮数（如 5-10 轮）， 快速验证整个流程是否正常工作。确认无误后，再使用更大的模型和更多的训练轮数进行正式训练。
📦

数据集格式

本模块使用 PyTorch 标准的 ImageFolder 格式组织数据集。这是最通用且易于理解的图像分类数据集组织方式， 只需将图片按类别放入不同的文件夹即可，无需额外的标注文件。

目录结构规范

方式一：单一数据集（自动划分验证集）

如果您只有一个数据集，系统会自动按照设定的比例（默认 20%）划分训练集和验证集：

my_dataset/ # ← 在界面中指向此目录 ├── class_A/ # 类别A（如：合格品） │ ├── image_001.jpg │ ├── image_002.jpg │ ├── image_003.png │ └── ... # 支持子文件夹，会递归扫描 ├── class_B/ # 类别B（如：缺陷品） │ ├── sample_001.jpg │ ├── sample_002.jpeg │ └── ... ├── class_C/ # 类别C │ └── ... └── class_D/ # 可以有任意多个类别 └── ...

方式二：预先划分训练/验证集

如果您已经手动划分了训练集和验证集，可以使用以下结构：

my_dataset/ ├── train/ # ← 在界面中指向此目录 │ ├── class_A/ │ │ ├── train_001.jpg │ │ └── ... │ ├── class_B/ │ │ └── ... │ └── class_C/ │ └── ... │ └── val/ # 系统会自动检测同级的 val 目录 ├── class_A/ │ └── ... ├── class_B/ │ └── ... └── class_C/ └── ...
💡
自动检测验证集
当您指向 train 目录时，系统会自动检测同级目录下是否存在 val 或 valid 或 validation 目录。 如果存在，将使用该目录作为验证集；否则将从训练数据中自动划分。

支持的图片格式

格式扩展名特点推荐场景
JPEG.jpg, .jpeg有损压缩，文件小通用场景，网络图片

类别命名规范

  • 文件夹名 = 类别名：系统将文件夹名称作为类别标签
  • 自动排序：类别按文件夹名的字母/数字顺序排序，依次分配索引（0, 1, 2...）
  • 字符支持：支持中文、英文、数字和下划线
  • 避免特殊字符：不要使用 / \ : * ? " < > | 等特殊字符
  • 大小写敏感：Cat 和 cat 会被视为不同类别

类别顺序控制

如果需要固定类别顺序，可以在文件夹名前添加数字前缀：

dataset/ ├── 00_background/ # 索引 0 ├── 01_defect_scratch/ # 索引 1 ├── 02_defect_stain/ # 索引 2 ├── 03_defect_crack/ # 索引 3 └── 04_normal/ # 索引 4

数据处理技巧

数据量建议

数据量是影响模型效果的关键因素。以下是不同场景的数据量建议：

场景每类最少推荐数量说明
快速验证10 张50+ 张仅用于测试流程是否正常，不追求效果
小样本学习50 张200+ 张依赖预训练权重迁移学习，适合数据稀缺场景
标准训练200 张500-1000 张可获得较好的分类效果
高精度需求1000 张5000+ 张工业级应用推荐，追求高准确率
大规模训练5000 张10000+ 张复杂分类任务，如细粒度分类

数据质量检查清单

✓
良好的数据特征
  • 图片清晰，目标可辨认
  • 各类别样本数量相对均衡
  • 覆盖不同光照、角度、背景
  • 标注准确，无错误分类
  • 与实际应用场景相符
✗
需要避免的问题
  • 图片模糊、过曝、欠曝
  • 类别严重不平衡（如 1:100）
  • 存在重复图片
  • 标注错误或模糊
  • 训练/测试数据分布不一致

类别不平衡处理

当各类别样本数量差异较大时（如正常:缺陷 = 100:1），会影响模型训练效果。以下是常用的解决方案：

1. 数据层面

  • 过采样（Oversampling）：复制少数类样本，或使用 SMOTE 等方法生成新样本
  • 欠采样（Undersampling）：减少多数类样本数量，适合数据量充足的场景
  • 数据增强：对少数类使用更多的数据增强策略
  • 收集更多数据：最根本的解决方案

2. 算法层面

  • 加权采样：训练时对少数类样本赋予更高的采样概率
  • 类别加权损失：在损失函数中对少数类赋予更高的权重
  • Focal Loss：自动降低易分类样本的权重，关注难分类样本
⚠️
验证集注意事项
验证集应保持与实际应用场景相似的数据分布，不建议在验证集上进行数据增强或重采样。 验证集的目的是评估模型在真实场景中的表现，过度处理会导致评估结果失真。

数据增强详解

数据增强是提升模型泛化能力的重要手段，通过对训练图片进行各种变换， 模拟不同的拍摄条件和场景变化，让模型学习到更加鲁棒的特征。

基础增强（默认启用）

以下增强策略默认开启，适用于大多数场景：

🔄
几何变换
  • 随机水平翻转：50% 概率左右翻转
  • 随机裁剪：随机裁剪后缩放到目标尺寸
  • 随机旋转：±15° 范围内随机旋转
  • 随机缩放：0.8-1.2 倍随机缩放
🎨
颜色变换
  • 亮度调整：±20% 亮度变化
  • 对比度调整：±20% 对比度变化
  • 饱和度调整：±20% 饱和度变化
  • 色相偏移：±10% 色相变化

高级增强（可选）

以下增强策略可在训练参数中开启，适用于数据量较少或需要提高鲁棒性的场景：

MixUp 数据增强

将两张图片按一定比例混合，同时混合它们的标签：

原理说明
# MixUp 公式
mixed_image = λ * image_A + (1-λ) * image_B
mixed_label = λ * label_A + (1-λ) * label_B

# 其中 λ 从 Beta(α, α) 分布中采样，α 通常设为 0.2 或 1.0
# 例如：λ=0.7 时，新图片包含 70% 的图片A 和 30% 的图片B

CutMix 数据增强

将一张图片的矩形区域替换为另一张图片的对应区域：

原理说明
# CutMix 操作
1. 随机选择一个矩形区域的位置和大小
2. 将图片A中该区域替换为图片B的相同区域
3. 标签按照替换面积比例混合

# 优势：保留了更多的局部特征信息
# 相比 MixUp，CutMix 在很多任务上效果更好

RandAugment

从一组预定义的增强操作中随机选择 N 个操作，每个操作使用相同的强度 M：

  • 操作集合包括：旋转、平移、剪切、亮度、对比度、锐化、Posterize、Solarize 等
  • 参数简单：只需调整 N（操作数量）和 M（操作强度）
  • 通常设置 N=2, M=9 效果较好

增强策略选择建议

场景推荐策略说明
数据充足（>5000/类）基础增强数据量足够，过度增强可能降低效果
数据适中（1000-5000/类）基础增强 + MixUp适度增强提升泛化能力
数据较少（<1000/类）基础增强 + CutMix + RandAugment强增强弥补数据不足
医疗/遥感等专业领域领域特定增强需要根据具体任务定制

⚠️
过拟合判断
当 Train Loss 持续下降 但 Val Loss 开始上升 时，说明模型开始过拟合。 此时应该：(1) 减小学习率；(2) 增加数据增强；(3) 使用更小的模型；(4) 增加 Dropout；(5) 收集更多数据。

调参技巧

学习率调整

学习率是最重要的超参数，直接影响训练效果和收敛速度。

📉
学习率过小的表现
  • Loss 下降非常缓慢
  • 训练曲线几乎是平直的
  • 需要大量 epochs 才能收敛
  • 最终效果可能不错，但效率低

解决方案：将学习率增大 2-10 倍

📈
学习率过大的表现
  • Loss 剧烈震荡，无法稳定下降
  • Loss 突然变成 NaN 或无穷大
  • 准确率上下波动，无法提升
  • 训练不稳定，难以复现

解决方案：将学习率减小 5-10 倍

Batch Size 与学习率的关系

经验法则：Batch Size 翻倍时，学习率也应该相应增大（线性缩放规则）

参考配置
# Batch Size 与学习率的推荐配置

Batch Size 8   →  Learning Rate 0.00025
Batch Size 16  →  Learning Rate 0.0005
Batch Size 32  →  Learning Rate 0.001    # 基准配置
Batch Size 64  →  Learning Rate 0.002
Batch Size 128 →  Learning Rate 0.004
Batch Size 256 →  Learning Rate 0.008

# 注意：当 Batch Size 很大时（>256），线性缩放可能过于激进
# 建议使用平方根缩放：lr = base_lr * sqrt(batch_size / 32)

模型转换概述

训练完成后，PyTorch 模型需要转换为优化的部署格式才能在生产环境中高效运行。 本模块支持转换到多种主流推理后端，每种后端都有其特定的优势和适用场景。

支持的目标格式

📦
ONNX (Open Neural Network Exchange)
开放神经网络交换格式，是模型转换的中间格式。 几乎所有推理框架都支持 ONNX，是跨平台部署的首选。
  • 跨平台、跨框架兼容
  • 可作为其他格式的转换源
  • 支持多种运行时优化
⚡
TensorRT (NVIDIA)
NVIDIA 官方推理优化器，专为 NVIDIA GPU 设计。 通过层融合、精度校准等技术实现极致性能。
  • NVIDIA GPU 上最快的推理速度
  • 支持 FP32/FP16/INT8 精度
  • 动态 Batch 支持
🔷
OpenVINO (Intel)
Intel 推理优化工具包，针对 Intel CPU/GPU/VPU 优化。 是边缘计算和 Intel 平台部署的首选。
  • Intel CPU 上性能优异
  • 支持 Intel 集成显卡
  • 支持 Neural Compute Stick
🟢
ONNX Runtime
微软开源的高性能推理引擎，支持多种硬件后端。 易于部署，跨平台兼容性好。
  • 跨平台：Windows/Linux/macOS
  • 支持 CPU/GPU/NPU 等多种设备
  • 与 ONNX 格式无缝集成

精度选项对比

精度模型大小推理速度精度损失显存占用适用场景
FP32 1x（基准） 1x（基准） 无 1x 开发调试、精度敏感场景
FP16 0.5x 1.5-2x 极小（<0.1%） 0.5x 推荐：大多数部署场景
INT8 0.25x 2-4x 小（0.5-2%） 0.25x 边缘设备、极致速度需求
Mixed 0.3-0.4x 2-3x 较小 0.3-0.4x 平衡精度和速度
💡
INT8 量化需要校准数据
INT8 量化需要提供校准数据集来计算最佳量化参数。校准数据应该：
  • 覆盖各个类别，具有代表性
  • 数量在 100-500 张之间
  • 可以使用训练集的子集

ONNX（Open Neural Network Exchange）是一种开放的模型表示格式， 几乎所有主流推理框架都支持。通常先将 PyTorch 模型导出为 ONNX， 再转换为其他特定后端的格式。

TensorRT 转换

TensorRT 是 NVIDIA 的深度学习推理优化器和运行时，通过层融合、 内核自动调优、精度校准等技术，在 NVIDIA GPU 上实现最佳推理性能。

转换参数

参数默认值说明
precisionFP16推理精度：FP32 / FP16 / INT8 / Mixed
workspace_gb4GPU 工作空间大小（GB），越大优化越好
dynamic_batchFalse是否启用动态 Batch Size
min_batch1最小 Batch Size
opt_batch4最优 Batch Size（用于优化）
max_batch16最大 Batch Size
calib_data-INT8 校准数据路径（INT8精度必需）

输出文件说明

converted/ ├── model_tensorrt_20240115.onnx # 中间 ONNX 文件 ├── model_tensorrt_fp16.engine # TensorRT 引擎（主要部署文件） └── model_tensorrt_fp16.yaml # 推理配置（包含类别映射等）

TensorRT 推理代码示例

python
import tensorrt as trt
import pycuda.driver as cuda
import pycuda.autoinit
import numpy as np
from PIL import Image
import yaml

# 加载配置
with open('model_tensorrt_fp16.yaml', 'r') as f:
    config = yaml.safe_load(f)

input_size = config['input_size']
mean = np.array(config['normalize_mean']).reshape(1, 3, 1, 1)
std = np.array(config['normalize_std']).reshape(1, 3, 1, 1)
class_names = config['class_names']

# 加载 TensorRT 引擎
logger = trt.Logger(trt.Logger.WARNING)
with open('model_tensorrt_fp16.engine', 'rb') as f:
    engine = trt.Runtime(logger).deserialize_cuda_engine(f.read())
context = engine.create_execution_context()

# 预处理函数
def preprocess(image_path):
    img = Image.open(image_path).convert('RGB')
    img = img.resize((input_size, input_size), Image.BICUBIC)
    img = np.array(img).astype(np.float32) / 255.0
    img = img.transpose(2, 0, 1)[np.newaxis, ...]  # HWC -> NCHW
    img = (img - mean) / std
    return img.astype(np.float32)

# 推理函数
def predict(image_path):
    # 预处理
    input_data = preprocess(image_path)
    
    # 分配 GPU 内存
    d_input = cuda.mem_alloc(input_data.nbytes)
    output = np.empty((1, len(class_names)), dtype=np.float32)
    d_output = cuda.mem_alloc(output.nbytes)
    
    # 执行推理
    cuda.memcpy_htod(d_input, input_data)
    context.execute_v2([int(d_input), int(d_output)])
    cuda.memcpy_dtoh(output, d_output)
    
    # 后处理
    probs = np.exp(output) / np.exp(output).sum(axis=1, keepdims=True)
    pred_idx = probs.argmax()
    
    return class_names[pred_idx], probs[0, pred_idx]

# 使用示例
label, confidence = predict('test.jpg')
print(f'预测结果: {label}, 置信度: {confidence:.4f}')

OpenVINO（Open Visual Inference and Neural Network Optimization）是 Intel 开发的 深度学习推理工具包，特别适合在 Intel CPU、集成 GPU 和 VPU 上进行高效推理。

OpenVINO 推理代码示例

python
from openvino.runtime import Core
import numpy as np
from PIL import Image
import yaml

# 加载配置
with open('converted/openvino/config.yaml', 'r') as f:
    config = yaml.safe_load(f)

input_size = config['input_size']
mean = np.array(config['normalize_mean']).reshape(1, 3, 1, 1)
std = np.array(config['normalize_std']).reshape(1, 3, 1, 1)
class_names = config['class_names']

# 初始化 OpenVINO
core = Core()
model = core.read_model('converted/openvino/model.xml')
compiled_model = core.compile_model(model, 'AUTO')  # 自动选择设备
infer_request = compiled_model.create_infer_request()

# 预处理
def preprocess(image_path):
    img = Image.open(image_path).convert('RGB')
    img = img.resize((input_size, input_size), Image.BICUBIC)
    img = np.array(img).astype(np.float32) / 255.0
    img = img.transpose(2, 0, 1)[np.newaxis, ...]
    img = (img - mean) / std
    return img.astype(np.float32)

# 推理
def predict(image_path):
    input_data = preprocess(image_path)
    infer_request.infer({0: input_data})
    output = infer_request.get_output_tensor(0).data
    
    probs = np.exp(output) / np.exp(output).sum(axis=1, keepdims=True)
    pred_idx = probs.argmax()
    
    return class_names[pred_idx], probs[0, pred_idx]

# 使用
label, confidence = predict('test.jpg')
print(f'预测: {label}, 置信度: {confidence:.4f}')

ONNX Runtime

ONNX Runtime 是微软开源的高性能推理引擎，支持多种硬件后端， 具有良好的跨平台兼容性，是通用部署场景的优秀选择。

支持的执行提供者（Execution Providers）

Provider硬件说明
CPUExecutionProviderCPU默认提供者，跨平台支持
CUDAExecutionProviderNVIDIA GPUGPU 加速推理
TensorrtExecutionProviderNVIDIA GPUTensorRT 优化
OpenVINOExecutionProviderIntel 设备Intel 优化
DirectMLExecutionProviderWindows GPUWindows 平台 GPU

ONNX Runtime 推理代码示例

python
import onnxruntime as ort
import numpy as np
from PIL import Image
import yaml

# 加载配置
with open('model_config.yaml', 'r') as f:
    config = yaml.safe_load(f)

input_size = config['input_size']
mean = np.array(config['normalize_mean']).reshape(1, 3, 1, 1)
std = np.array(config['normalize_std']).reshape(1, 3, 1, 1)
class_names = config['class_names']

# 创建推理会话（自动选择最佳执行提供者）
providers = ['CUDAExecutionProvider', 'CPUExecutionProvider']
session = ort.InferenceSession('model.onnx', providers=providers)
input_name = session.get_inputs()[0].name
output_name = session.get_outputs()[0].name

print(f"使用设备: {session.get_providers()[0]}")

# 预处理
def preprocess(image_path):
    img = Image.open(image_path).convert('RGB')
    img = img.resize((input_size, input_size), Image.BICUBIC)
    img = np.array(img).astype(np.float32) / 255.0
    img = img.transpose(2, 0, 1)[np.newaxis, ...]
    img = (img - mean) / std
    return img.astype(np.float32)

# 推理
def predict(image_path):
    input_data = preprocess(image_path)
    output = session.run([output_name], {input_name: input_data})[0]
    
    probs = np.exp(output) / np.exp(output).sum(axis=1, keepdims=True)
    pred_idx = probs.argmax()
    
    return class_names[pred_idx], probs[0, pred_idx]

# 批量推理
def predict_batch(image_paths, batch_size=16):
    results = []
    for i in range(0, len(image_paths), batch_size):
        batch_paths = image_paths[i:i+batch_size]
        batch_inputs = np.vstack([preprocess(p) for p in batch_paths])
        outputs = session.run([output_name], {input_name: batch_inputs})[0]
        
        probs = np.exp(outputs) / np.exp(outputs).sum(axis=1, keepdims=True)
        for j, path in enumerate(batch_paths):
            pred_idx = probs[j].argmax()
            results.append({
                'path': path,
                'label': class_names[pred_idx],
                'confidence': probs[j, pred_idx]
            })
    return results

# 使用示例
label, conf = predict('test.jpg')
print(f'预测: {label}, 置信度: {conf:.4f}')

API 接口调用

通过 Python 代码直接调用推理功能。

python
import torch
import timm
from PIL import Image
from torchvision import transforms

# 加载模型
checkpoint = torch.load('best_model.pth', map_location='cuda')
model = timm.create_model(checkpoint['model_name'], 
                          num_classes=checkpoint['num_classes'], 
                          pretrained=False)
model.load_state_dict(checkpoint['model'])
model.cuda().eval()

# 预处理
input_size = checkpoint['input_size']
transform = transforms.Compose([
    transforms.Resize(int(input_size * 1.14)),
    transforms.CenterCrop(input_size),
    transforms.ToTensor(),
    transforms.Normalize(mean=checkpoint['normalize_mean'],
                        std=checkpoint['normalize_std']),
])

# 推理
def predict(image_path):
    img = Image.open(image_path).convert('RGB')
    tensor = transform(img).unsqueeze(0).cuda()
    with torch.no_grad():
        output = model(tensor)
        probs = torch.softmax(output, dim=1)
        idx = probs.argmax().item()
    return checkpoint['idx_to_class'][idx], probs[0, idx].item()

label, conf = predict('test.jpg')
print(f'{label}: {conf:.4f}')

常见问题

Q
CUDA out of memory 错误

解决：减小 Batch Size、使用更小模型、减小图像尺寸、启用混合精度

Q
准确率卡在随机水平

解决：降低学习率 10 倍、检查数据标签、确认预训练权重已加载

Q
如何使用自己的数据集

将图片按类别放入文件夹，然后输入文件夹路径即可。

Q
训练过拟合

解决：增加数据增强、使用更小模型、增加 Dropout、收集更多数据
