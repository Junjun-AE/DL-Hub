异常检测模块

基于 PatchCore 的工业级无监督异常检测解决方案。仅需良品数据即可训练， 自动学习正常样本特征分布，精准识别各类异常缺陷，输出像素级热力图定位。

🏗️ 算法：PatchCore
🎯 无监督学习
📊 像素级定位
📅 更新：2025年1月
📋

功能概览

异常检测模块是 DL-Hub 的核心组件之一，专为工业质检场景设计。与传统的有监督缺陷检测不同， 异常检测只需要良品（正常）样本进行训练，无需收集和标注各种缺陷类型， 即可检测出任何偏离正常模式的异常。

🌟
核心优势：在工业场景中，良品容易获取，而缺陷样本稀少且种类繁多。 异常检测可以解决"长尾问题"——即使是从未见过的新型缺陷也能被检出！

核心特性

🎯

仅需良品数据

无需收集和标注缺陷样本，只需提供良品图像即可完成训练，大幅降低数据准备成本。

🔬

检测未知缺陷

基于"异常=偏离正常"的原理，可以检测任何类型的异常，包括从未见过的新型缺陷。

🗺️

像素级定位

输出异常热力图，精确标注异常区域位置，支持可视化展示和进一步分析。

⚡

训练速度快

无需迭代训练，一次性提取特征并构建索引，训练过程通常只需几分钟。

🎚️

灵活阈值调节

归一化分数输出（0-100），可根据实际需求灵活调整检测敏感度。

🚀

高效部署

支持导出为 ONNX、TensorRT 格式，可在 GPU/CPU 上高效推理。

适用场景

  • 工业表面检测：金属划痕、布料瑕疵、玻璃裂纹、塑料缺陷
  • PCB/半导体：焊点异常、元件缺失、线路断裂、晶圆缺陷
  • 纹理检测：布料、皮革、木材等纹理产品的异常检测
  • 食品医药：药片、胶囊、食品包装的外观检测
  • 3C产品：手机屏幕、笔记本外壳、精密零件的质检
🚀

快速上手

只需4个步骤，即可完成从数据准备到异常检测的完整流程。整个训练过程通常只需几分钟！

1
准备良品数据
收集正常（良品）样本图像，放入 good/ 文件夹。 建议至少 50-100 张，覆盖正常样本的各种变化（光照、角度等）。
2

在"数据配置"页面设置数据集路径、选择场景模板或手动调整参数。 推荐使用"通用检测"模板作为起点。

点击"开始训练"，系统会自动提取特征、构建 Memory Bank、校准阈值。 训练过程通常只需 2-10 分钟。
4
批量检测
切换到"批量检测"页面，选择训练好的模型和待检测图像文件夹， 设置阈值后开始检测。输出包含分数、判定结果和热力图。
💡
快速验证：如果有少量异常样本，可以放入 defect/ 文件夹， 系统会在评估阶段计算检测指标，帮助你选择最佳阈值。

什么是异常检测

在深入了解 PatchCore 算法之前，我们先来理解"异常检测"这个任务本身。

有监督 vs 无监督

❌ 有监督缺陷检测（传统方法）
  • 需要收集大量缺陷样本并标注
  • 缺陷类型必须预先定义（划痕、污点、裂纹...）
  • 无法检测未见过的缺陷类型
  • 缺陷样本难以收集（稀有、成本高）
  • 数据不平衡问题严重
✅ 无监督异常检测（本模块）
  • 仅需良品数据即可训练
  • 无需预定义缺陷类型
  • 可检测任何异常，包括未知缺陷
  • 良品容易获取，数据准备成本低
  • 天然解决数据不平衡问题

核心思想：学习"正常"

💡 异常检测的基本原理

异常检测的核心思想是：学习正常样本的特征分布，然后将偏离这个分布的样本判定为异常。

想象你是一个质检员，每天检查同一种产品。经过一段时间，你对"正常"的产品长什么样已经了然于胸。 当出现一个"不太对劲"的产品时，即使你说不清具体哪里有问题，你也能感觉到它与正常产品不同。

异常检测算法就是在模拟这个过程——通过大量正常样本学习"正常"的模式， 然后用数学方法量化"与正常的偏离程度"。

为什么需要异常检测？

  • 长尾问题：工业缺陷种类繁多，某些缺陷极其罕见，难以收集足够样本
  • 未知缺陷：新产品上线时，可能出现设计阶段未预料到的缺陷类型
  • 快速部署：只需良品数据，可以快速建立检测系统
  • 通用性强：同一算法可应用于各种产品，无需针对每种缺陷单独训练
🧠

PatchCore 算法简介

PatchCore 是目前工业异常检测领域最先进、应用最广泛的算法之一， 由 Amazon 于 2022 年提出，在 MVTec AD 等权威基准上取得了 SOTA 性能。

📚
论文：《Towards Total Recall in Industrial Anomaly Detection》
PatchCore 的核心创新是使用预训练 CNN 提取局部特征，结合 Memory Bank 进行高效异常检测。

为什么 PatchCore 效果好？

🎓

预训练知识迁移

使用 ImageNet 预训练的 CNN（如 WideResNet50）提取特征，继承了强大的视觉表示能力。

🧩

局部特征建模

不是对整张图像建模，而是对每个局部区域（Patch）单独建模，提高了定位精度。

🎯

CoreSet 智能采样

通过贪心采样保留最具代表性的特征，在保持精度的同时大幅减少存储和计算量。

🔍

特征提取详解

特征提取是 PatchCore 的第一步，也是最关键的一步。它决定了模型能够"看到"图像中的什么信息。

预训练 Backbone

🧠 为什么使用预训练模型？

深度神经网络（如 ResNet）在 ImageNet 上经过大规模训练后，已经学会了丰富的视觉特征表示—— 从边缘、纹理到形状、语义，各个层级的特征都能被很好地提取。

我们直接复用这些预训练的特征提取能力，而不是从头训练。 这就是为什么 PatchCore 只需要少量良品数据就能工作得很好。

多层特征融合

PatchCore 默认从 Backbone 的 Layer2 和 Layer3 提取特征并融合：

层级 特征维度 感受野 特征类型
Layer2 512-D 较小 局部纹理、边缘细节
Layer3 1024-D 较大 中层语义、形状结构
融合后 1536-D 多尺度 综合特征表示
💡
为什么不用更深的层？
更深的层（Layer4、Layer5）具有更强的语义信息，但空间分辨率更低， 不利于精确定位。Layer2-3 在"局部细节"和"全局语义"之间取得了很好的平衡。

Patch 特征

🧩 什么是 Patch 特征？

以输入尺寸 256×256 为例，经过 Layer2 后特征图尺寸为 32×32。 这意味着图像被划分为 32×32 = 1024 个 Patch，每个 Patch 对应原图的一小块区域。

每个 Patch 都有一个 1536 维的特征向量，描述了这个局部区域的视觉特性。 这就是 PatchCore 中"Patch"的含义——基于局部区域进行异常检测。

一张图像的 Patch 数量
N_patches = (H / 8) × (W / 8) = 32 × 32 = 1024 个 Patch
💾

Memory Bank 详解

Memory Bank（记忆库）是 PatchCore 的核心数据结构，存储了所有正常样本的 Patch 特征。

Memory Bank 是什么？

💾 特征库的本质

Memory Bank 本质上是一个大型特征向量集合：

  • 训练时：收集所有良品图像的所有 Patch 特征，存入 Memory Bank
  • 推理时：将测试图像的 Patch 特征与 Memory Bank 中的特征进行比对
  • 如果某个 Patch 特征与 Memory Bank 中所有特征都"距离较远"，则判定为异常

规模计算示例

计算示例 
# 假设有 100 张训练图像，输入尺寸 256×256
训练图像数: 100
每张图像 Patch 数: 32 × 32 = 1,024
总 Patch 数: 100 × 1,024 = 102,400
特征维度: 1,536

# Memory Bank 原始大小
102,400 × 1,536 × 4字节(float32) = 629 MB

# 这就是为什么需要 CoreSet 采样来压缩！

PCA 降维（可选）

当特征维度较高时（如 1536-D），可以使用 PCA 进行降维以加速后续计算：

  • 默认目标维度：256-D
  • 方差保留：通常保留 99.5% 的方差信息
  • 效果：计算速度提升 3-5 倍，精度几乎无损
🎯

CoreSet 采样详解

CoreSet 采样是 PatchCore 的关键创新之一，它解决了 Memory Bank 过大导致的存储和计算问题。

为什么需要采样？

⚠️ 直接使用全部特征的问题

如果直接存储所有 Patch 特征到 Memory Bank：

  • 存储爆炸：1000 张图像 → 约 6GB 特征数据
  • 搜索缓慢：KNN 搜索时间与 Memory Bank 大小成正比
  • 冗余信息：很多 Patch 特征高度相似，存在大量冗余

k-Center Greedy 算法

CoreSet 使用贪心最远点采样（k-Center Greedy）算法来选择代表性特征：

KNN 搜索详解

KNN（K-Nearest Neighbors，K近邻）搜索是异常检测的核心步骤—— 通过计算测试特征到 Memory Bank 中最近邻的距离来判断异常程度。

异常检测原理

📐 距离 = 异常程度

核心思想非常直观：

  • 正常样本：其特征与 Memory Bank 中的正常特征距离较近
  • 异常样本：其特征与所有正常特征都距离较远

因此，最近邻距离越大，异常可能性越高。

异常分数计算
Anomaly_Score(patch) = min_distance(patch, Memory_Bank)

Faiss 加速

当 Memory Bank 较大时，暴力搜索效率低下。我们使用 Facebook 开源的 Faiss 库进行加速：

索引类型 适用规模 速度 精度
Flat < 5,000 基准 精确
IVFFlat 5,000 - 50,000 快 3-5x 接近精确
IVFPQ > 50,000 快 10-50x 近似

K 值选择

参数 K 表示取多少个近邻来计算距离，默认 K=9：

  • K=1：只看最近邻，对噪声敏感
  • K=5-9：推荐范围，平衡稳定性和敏感度
  • K>15：过于平滑，可能漏检小异常
📊

分数计算与归一化

PatchCore 输出两种结果：像素级异常图和图像级分数。

异常图生成

1
计算 Patch 距离
每个 Patch 的最近邻距离构成一个 32×32 的距离矩阵
2
上采样
使用双线性插值将距离矩阵上采样到原图尺寸
3
高斯平滑
应用高斯滤波使热力图更平滑，减少噪点（σ=4）

分数归一化

原始距离值没有固定范围，不便于设置阈值。我们使用百分位数归一化将分数映射到 0-100：

归一化公式
Normalized_Score = (Raw_Score - P1) / (P99 - P1) × 100

其中 P1 和 P99 是训练集分数的第 1 和第 99 百分位数。这样：

  • 分数 0-50：与正常样本相似，大概率正常
  • 分数 50-100：超出正常范围，可能异常
  • 分数 >100：严重异常（原始分数超过 P99）

图像级分数

图像级分数是异常图中的最大值，代表整张图像的异常程度：

图像级分数
Image_Score = max(Anomaly_Map)

💡

数据准备建议

良品数据质量

✅

覆盖正常变化

收集不同光照、角度、位置的正常样本，让模型学习"正常"的完整分布。

✅

确保无异常

良品数据中不能混入异常样本！否则模型会把异常也学成"正常"。

❌

避免过于单一

如果所有良品都完全相同，模型可能对正常变化过于敏感，导致误报。

数据量建议

场景 建议良品数量 说明
简单纹理（布料、金属表面） 50-100 张 纹理规律性强，少量样本即可覆盖
结构产品（PCB、零件） 100-200 张 结构复杂度适中
复杂场景（多变光照、位置） 200-500 张 需要覆盖各种正常变化

训练过程

PatchCore 的训练不是传统的梯度下降迭代，而是一次性的特征提取和索引构建过程。

训练阶段

阶段 耗时占比 说明
1. 初始化 ~5% 加载 Backbone、创建 DataLoader
2. 特征提取 ~40% 提取所有图像的 Patch 特征
3. PCA 降维 ~10% 使用增量 PCA 进行降维
4. CoreSet 采样 ~20% 贪心采样保留代表性特征
5. 构建索引 ~10% 构建 Faiss KNN 索引
6. 阈值校准 ~10% 计算归一化参数和预设阈值
7. 模型导出 ~5% 保存模型和配置文件
⏱️
训练时间参考：
100 张图像 → 约 2-3 分钟
500 张图像 → 约 5-10 分钟

阈值设置

阈值是决定"多异常才算异常"的关键参数。归一化分数范围为 0-100， 分数 ≥ 阈值 判定为异常。

阈值指南

30-40
高召回
漏检少，误报多
安全关键场景
40-55
平衡
推荐日常使用
精度召回平衡
55-70
高精度
误报少，可能漏检
减少人工复核
70+
极严格
仅检测明显异常
特殊场景使用

如何选择阈值？

  1. 先用默认值 50 进行测试
  2. 如果误报太多（正常被判为异常）→ 提高阈值
  3. 如果漏检太多（异常被判为正常）→ 降低阈值
  4. 如果有异常样本，可以在"评估验证"页面查看 F1 曲线，找到最优阈值
📊

效果评估

如果提供了异常样本（defect/ 文件夹），系统会自动计算评估指标。

核心指标

  • Precision（精确率）：预测为异常的样本中，真正异常的比例。精确率高 = 误报少
  • Recall（召回率）：所有异常样本中，被正确检出的比例。召回率高 = 漏检少
  • F1 Score：精确率和召回率的调和平均，综合评价指标
  • AUROC：不同阈值下的综合性能，越接近 1 越好
F1 Score 计算
F1 = 2 × (Precision × Recall) / (Precision + Recall)

常见问题

误报太多（正常被判为异常）怎么办？
1. 提高阈值：从 50 调整到 60-70
2. 增加训练数据：添加更多良品样本，尤其是容易误报的类型
3. 检查数据质量：确保训练集中没有混入异常样本
漏检太多（异常没有被检出）怎么办？
1. 降低阈值：从 50 调整到 30-40
2. 增大图像尺寸：如果异常很小，尝试 384 或 512
3. 增加采样率：将 coreset_ratio 从 1% 提高到 5-10%
训练速度很慢怎么办？
1. 使用更小的 image_size（如 224）
2. 使用轻量级 backbone（如 ResNet18）
3. 减少 coreset_ratio（如 0.5%）
4. 确保使用 GPU 并开启 FP16
内存不足怎么办？
1. 减少训练图像数量
2. 减小 image_size
3. 开启 use_fp16_features（默认开启）
4. 减少 feature_chunk_size
如何检测多种产品？
每种产品需要单独训练一个模型。异常检测是基于"正常"的定义， 不同产品的"正常"不同，不能混合训练。
