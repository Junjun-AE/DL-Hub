目标检测模块

基于 Ultralytics YOLO 框架构建的专业目标检测训练平台，支持 YOLOv5、YOLOv8、YOLOv11 三大系列共 15 个预训练模型， 涵盖从数据标注、模型训练、模型转换到生产部署的完整工作流程。无论是工业质检、智能安防还是自动驾驶场景， 都能快速构建高精度的目标检测系统。

🧠15 个 YOLO 模型
⚡TensorRT 加速
🎨LabelMe 标注支持
📦多后端部署
📊实时训练监控
📖

概述

目标检测模块是 DL-Hub 深度学习平台的核心组件之一，专为工业级目标检测任务设计。 该模块基于 Ultralytics YOLO 框架构建，集成了 YOLOv5、YOLOv8、YOLOv11 三大系列的预训练模型， 涵盖从超轻量级边缘部署到高精度服务器推理的全场景需求。

本模块原生支持 LabelMe 标注格式，自动完成到 YOLO 格式的转换，无需手动处理数据格式问题。 通过直观的 Web 界面，您可以轻松配置训练参数、监控训练进度（包括 Loss 曲线、mAP 曲线）、 转换模型格式，并进行批量推理测试。

核心特性

🧠
三代 YOLO 模型
支持 YOLOv5/v8/v11 三个系列共 15 个模型，从 1.9M 到 86.7M 参数量全覆盖
📝
LabelMe 原生支持
直接使用 LabelMe 标注的 JSON 文件，系统自动转换为 YOLO 格式
📈
实时训练监控
Loss 曲线、mAP 曲线实时更新，支持训练中断与恢复
⚡
多后端转换
一键转换至 TensorRT、ONNX Runtime、OpenVINO，支持 INT8 量化

快速入门

本节将指导您在 10 分钟内完成第一个目标检测模型的训练。 我们将使用一个简单的示例数据集来演示完整的工作流程。

1
准备 LabelMe 标注数据

使用 LabelMe 工具标注您的图像数据，确保：

  • 所有标注使用矩形框（Rectangle）
  • 标签（Label）使用数字 ID，如 0, 1, 2, 3...
  • 图像文件夹和 JSON 文件夹可以相同或不同
my_dataset/ ├── images/ # 图像文件夹 │ ├── 001.jpg │ ├── 002.jpg │ └── ... └── jsons/ # JSON标注文件夹 ├── 001.json ├── 002.json └── ...
2
启动目标检测模块

通过 DL-Hub 主界面或命令行启动：

bash
# 方式一：通过 DL-Hub 主界面
# 点击「目标检测」任务卡片，选择「启动应用」

# 方式二：命令行启动
cd model_image_detection
python app.py --port 7860
3
配置训练参数

在 Web 界面的「模型训练」标签页中进行以下配置：

  1. 图像文件夹路径：输入包含训练图像的文件夹路径
  2. JSON 标注文件夹路径：输入包含 LabelMe JSON 标注的文件夹路径
  3. 模型选择：选择模型系列（推荐 YOLOv8）和规模（推荐「中」）
  4. 训练参数：首次使用可保持默认值：
    • 训练轮数（Epochs）：建议 50-100 轮
    • 批次大小（Batch Size）：根据显存调整，通常 8-16
    • 学习率：默认 0.01，通常无需修改
    • 图像尺寸：默认 640
  5. GPU 选择：如有多张显卡，可选择指定 GPU
4
开始训练并监控

点击「开始训练」按钮后，您可以实时监控训练进度：

  • 训练日志：显示每个 epoch 的损失值、学习率等信息
  • Loss 曲线：实时绘制 Box Loss 和 Cls Loss 的变化趋势
  • mAP 曲线：展示 mAP@50 和 mAP@50:95 的提升过程
  • 最佳模型：系统会自动保存验证 mAP 最高的模型

训练过程中可以随时点击「停止训练」按钮安全中断，系统会保存当前检查点。

5
使用训练好的模型

训练完成后，您可以：

  • 批量推理：切换到「批量推理」标签页，对新图片进行目标检测
  • 模型转换：切换到「模型转换」标签页，将模型转换为 TensorRT/ONNX 等部署格式
  • 导出模型：在输出目录中找到 best_model.pt 和 last_model.pt 文件
✅
快速验证建议
首次使用时，建议选择较小的模型（如 YOLOv8n）和较少的训练轮数（如 10-20 轮）， 快速验证整个流程是否正常工作。确认无误后，再使用更大的模型和更多的训练轮数进行正式训练。
📦

LabelMe 标注格式

本模块原生支持 LabelMe 标注工具生成的 JSON 格式，无需手动转换为 YOLO 格式。 系统会在训练开始时自动完成格式转换。

目录结构要求

您需要准备两个文件夹：一个包含图像，一个包含 JSON 标注文件。它们可以是同一个文件夹，也可以分开存放。

dataset/ ├── images/ # 图像文件夹 │ ├── img_001.jpg # 支持 jpg, jpeg, png, bmp, webp, tiff │ ├── img_002.png │ ├── img_003.jpg # 这张图没有对应的JSON，将作为负样本 │ └── ... └── annotations/ # JSON标注文件夹 ├── img_001.json # 文件名需与图像对应（不含扩展名） ├── img_002.json └── ...

JSON 文件格式

每个 JSON 文件包含一张图像的所有标注信息：

json
{
  "version": "5.0.1",
  "flags": {},
  "shapes": [
    {
      "label": "0",           // 类别ID（必须是数字字符串）
      "points": [
        [100.5, 200.3],       // 左上角坐标 [x1, y1]
        [300.8, 450.6]        // 右下角坐标 [x2, y2]
      ],
      "group_id": null,
      "shape_type": "rectangle",  // 必须是 rectangle
      "flags": {}
    },
    {
      "label": "1",           // 另一个目标，类别ID为1
      "points": [
        [50.0, 100.0],
        [150.0, 200.0]
      ],
      "shape_type": "rectangle"
    }
  ],
  "imagePath": "img_001.jpg",
  "imageData": null,          // 建议设为 null 以减小文件体积
  "imageHeight": 1080,        // 图像高度（像素）
  "imageWidth": 1920          // 图像宽度（像素）
}

标注规范

⚠️
关键要求
  • 标签必须是数字：类别 ID 使用 0, 1, 2, 3... 的数字字符串，不支持文字标签
  • 只支持矩形框：shape_type 必须是 "rectangle"，多边形和其他形状会被忽略
  • 坐标使用像素值：points 中的坐标是图像中的实际像素位置
  • 类别 ID 连续性：建议使用从 0 开始的连续整数（0, 1, 2, 3...）

负样本处理

系统支持将没有对应 JSON 文件的图像作为负样本（也称为良品或背景样本）。 这对于工业质检场景特别有用，可以帮助模型学习区分正常样品和缺陷样品。

✅
负样本最佳实践
建议负样本（无标注图像）占总样本的 10-30%。过多的负样本可能导致模型过于保守， 而过少的负样本可能导致误检率上升。
💡

数据标注技巧

标注质量要求

✅
正确的标注方式
  • 边界框紧贴目标边缘，不留太多空白
  • 目标被部分遮挡时，标注可见部分
  • 同一目标只标注一次
  • 标注所有符合条件的目标，不遗漏
  • 使用一致的标注标准
❌
应避免的问题
  • 边界框过大，包含大量背景
  • 边界框过小，裁掉目标边缘
  • 同一目标重复标注
  • 漏标明显的目标
  • 类别标注错误或不一致

数据量建议

任务复杂度每类建议数量说明
简单目标（如车辆、人脸）200-500 张目标形态规则，类内差异小
中等目标（如缺陷检测）500-2000 张目标形态多样，需要更多样本
复杂目标（如细粒度分类）2000+ 张类间差异小，需要大量样本区分

格式自动转换

训练开始时，系统会自动将 LabelMe JSON 格式转换为 YOLO TXT 格式。 转换过程完全透明，无需用户干预。

转换流程

转换示意
LabelMe JSON 格式                    YOLO TXT 格式
─────────────────                    ─────────────────
{                                    
  "shapes": [                        # class_id x_center y_center width height
    {                                # (归一化坐标，范围 0-1)
      "label": "0",                  
      "points": [                    0 0.2083 0.3009 0.1042 0.2315
        [100.5, 200.3],      →      1 0.0521 0.1389 0.0521 0.0926
        [300.8, 450.6]               
      ]                              
    },                               
    ...                              
  ],                                 
  "imageWidth": 1920,                
  "imageHeight": 1080                
}

转换后的目录结构

output/yolov8m_20260130_120000/ ├── dataset/ # 自动生成的YOLO格式数据集 │ ├── images/ │ │ ├── train/ # 训练集图像 │ │ └── val/ # 验证集图像 │ ├── labels/ │ │ ├── train/ # 训练集标注（.txt） │ │ └── val/ # 验证集标注（.txt） │ ├── data.yaml # YOLO配置文件 │ └── class_info.txt # 类别信息说明 └── ...

data.yaml 配置文件

自动生成的配置文件包含数据集路径和类别信息：

yaml
# 自动生成的 YOLO 数据集配置
path: /path/to/output/yolov8m_20260130_120000/dataset
train: images/train
val: images/val

# 类别数量
nc: 5

# 类别名称（使用数字ID）
names:
  - '0'
  - '1'
  - '2'
  - '3'
  - '4'

learning_rate float 0.01 初始学习率。SGD 推荐 0.01，Adam 推荐 0.001
optimizer str "SGD" 优化器。可选 SGD、Adam、AdamW

高级参数

参数类型默认值说明
patience int 50 早停耐心值。验证指标连续 N 轮不提升则停止
weight_decay float 0.0005 权重衰减（L2 正则化）
momentum float 0.937 SGD 动量
warmup_epochs float 3.0 学习率预热轮数
cos_lr bool True 是否使用余弦退火学习率
val_split float 0.2 验证集比例。设为 0 则训练集同时用于验证

数据增强参数

参数类型默认值说明
mosaic float 1.0 Mosaic 增强概率。四图拼接，有效增加小目标
mixup float 0.0 MixUp 增强概率。图像混合
fliplr float 0.5 水平翻转概率
⚠️
小数据集注意事项
当数据集较小（<500 张）时，建议： ① 使用较小的模型（n/s）避免过拟合； ② 增大数据增强强度（mosaic=1.0, mixup=0.5）； ③ 减小 batch_size 以获得更多更新步数； ④ 适当增加 epochs 但监控过拟合。

调参技巧

常见问题与解决方案

1
训练 Loss 不下降

可能原因：学习率过大或过小、数据标注有问题、模型过于简单

解决方案：

  • 降低学习率（如从 0.01 降到 0.001）
  • 检查数据标注是否正确
  • 增加 warmup_epochs
  • 尝试更大的模型
2
验证 mAP 上升后下降（过拟合）

可能原因：数据量不足、模型过大、数据增强不够

解决方案：

  • 增加数据增强强度（mixup=0.5, mosaic=1.0）
  • 使用更小的模型
  • 增加权重衰减（weight_decay=0.001）
  • 减少训练 epochs，依靠早停
3
小目标检测效果差

可能原因：输入分辨率不够、小目标样本不足

解决方案：

  • 增大 img_size（如 1280）
  • 使用 Mosaic 增强（mosaic=1.0）
  • 增加小目标的标注样本
  • 考虑使用切图策略
4
CUDA out of memory

解决方案：

  • 减小 batch_size（如从 16 减到 8）
  • 减小 img_size（如从 640 减到 512）
  • 使用更小的模型（如 n 或 s）
  • 关闭 Mosaic 增强

评估指标详解

核心指标

指标含义计算方式理想值
mAP@50 IoU=0.5 时的平均精度 所有类别 AP@50 的平均值 越高越好，通常 > 0.8
mAP@50:95 IoU=0.5:0.95 的平均精度 不同 IoU 阈值下 AP 的平均 越高越好，通常 > 0.5
Precision 精确率 TP / (TP + FP) 高精度场景需 > 0.9
Recall 召回率 TP / (TP + FN) 低漏检场景需 > 0.9
Box Loss 边界框回归损失 定位误差 越低越好
Cls Loss 分类损失 分类误差 越低越好

IoU（交并比）说明

IoU（Intersection over Union）是衡量预测框与真实框重叠程度的指标：

公式
IoU = 预测框 ∩ 真实框 / 预测框 ∪ 真实框

IoU = 0.5  →  预测框与真实框重叠 50%
IoU = 0.75 →  预测框与真实框重叠 75%
IoU = 1.0  →  预测框与真实框完全重合

模型转换概述

训练完成后，您可以将 PyTorch 模型转换为高性能推理引擎格式， 以获得更快的推理速度和更低的资源消耗。

支持的后端

后端适用场景支持精度硬件要求
TensorRT NVIDIA GPU 高性能推理 FP32, FP16, INT8 NVIDIA GPU
ONNX Runtime 跨平台通用推理 FP32, FP16 CPU/GPU 均可
OpenVINO Intel 硬件优化 FP32, FP16, INT8 Intel CPU/GPU/VPU

精度模式说明

FP32
单精度浮点

保持原始精度，无损转换。适合对精度要求极高的场景。

FP16
半精度浮点

推荐选择。精度损失极小，速度提升 50-100%，显存减半。

INT8
8位整数

最大加速。需要校准数据，可能有少量精度损失。

Mixed
混合精度

敏感层用 FP16，其他层用 INT8。平衡精度和速度。

⚡

TensorRT 转换

TensorRT 是 NVIDIA 的高性能深度学习推理优化器，可提供最快的 GPU 推理速度。

转换步骤

  1. 切换到「模型转换」标签页
  2. 选择要转换的模型文件（.pt）
  3. 目标后端选择 tensorrt
  4. 精度模式选择 fp16（推荐）
  5. 设置 Workspace 大小（默认 4GB）
  6. 点击「开始转换」

INT8 量化校准

如果选择 INT8 精度，需要提供校准数据集：

  • 准备 100-500 张代表性图片
  • 无需标注文件，只需图片
  • 图片应覆盖实际使用场景的分布
💡
TensorRT 版本兼容性
生成的 TensorRT 引擎与特定 GPU 架构和 TensorRT 版本绑定。 在不同硬件或版本上部署时，需要重新转换。
🟢

ONNX Runtime 转换

ONNX Runtime 提供跨平台的推理支持，适合需要在多种硬件上部署的场景。

转换步骤

  1. 目标后端选择 onnxruntime
  2. 精度模式选择 fp16 或 fp32
  3. 点击「开始转换」

import onnxruntime as ort
import numpy as np
import cv2

# 创建推理会话
session = ort.InferenceSession(
    "best_model.onnx",
    providers=['CUDAExecutionProvider', 'CPUExecutionProvider']
)

# 预处理图像
def preprocess(image_path, input_size=640):
    img = cv2.imread(image_path)
    img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
    img = cv2.resize(img, (input_size, input_size))
    img = img.astype(np.float32) / 255.0
    img = np.transpose(img, (2, 0, 1))
    img = np.expand_dims(img, axis=0)
    return img

# 推理
input_tensor = preprocess("test.jpg")
outputs = session.run(None, {"images": input_tensor})

from ultralytics import YOLO
import cv2

# 加载模型
model = YOLO('output/yolov8m_xxx/weights_model_importer/best_model.pt')

# 单张图片推理
results = model.predict(
    source='test.jpg',
    conf=0.25,      # 置信度阈值
    iou=0.45,       # NMS IoU阈值
    save=False,     # 不自动保存
    verbose=False
)

# 处理结果
for result in results:
    boxes = result.boxes
    for box in boxes:
        # 获取坐标 [x1, y1, x2, y2]
        x1, y1, x2, y2 = box.xyxy[0].cpu().numpy()
        # 获取置信度
        confidence = box.conf[0].cpu().numpy()
        # 获取类别ID
        class_id = int(box.cls[0].cpu().numpy())
        
        print(f"类别: {class_id}, 置信度: {confidence:.4f}, 坐标: [{x1:.0f}, {y1:.0f}, {x2:.0f}, {y2:.0f}]")

# 批量推理
results = model.predict(

常见问题

Q
标签必须是数字吗？不能用文字吗？

答：是的，目前系统要求标签使用数字 ID（0, 1, 2...）。这是为了确保与 YOLO 格式的兼容性。您可以在后处理时将数字 ID 映射为有意义的文字标签。

Q
为什么训练时显示 mAP 为 0？

答：可能原因：① 数据标注格式不正确；② 类别 ID 与实际数据不匹配；③ 训练刚开始，模型还没学到有效特征。建议检查转换后的 YOLO 标注文件是否正确。

Q
如何处理类别不平衡问题？

答：① 对少数类别进行过采样；② 使用数据增强增加少数类别的变体；③ 调整 loss 权重（需要修改代码）；④ 收集更多少数类别的样本。

Q
TensorRT 转换失败怎么办？

答：① 检查 CUDA 和 TensorRT 版本是否匹配；② 尝试减小 Workspace 大小；③ 先转换为 ONNX 格式确认模型正常；④ 查看详细错误日志定位问题。

Q
如何提高小目标检测效果？

答：① 增大输入尺寸（如 1280）；② 使用 Mosaic 数据增强；③ 增加小目标的训练样本；④ 使用切图策略将大图切成小块分别检测。
