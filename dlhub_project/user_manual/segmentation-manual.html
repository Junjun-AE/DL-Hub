语义分割模块

基于 SegFormer 的工业级语义分割解决方案，支持像素级精确分割、多类别标注、 滑动窗口大图推理，输出 LabelMe 格式标注，无缝对接工业质检流程。

🏗️ 架构：SegFormer (MiT-B0~B5)
📊 6 种模型规格
🔧 框架：MMSegmentation
📅 更新：2025年1月
📋

功能概览

语义分割模块是 DL-Hub 的核心组件之一，专为工业缺陷检测、医学图像分析等像素级标注任务设计。 基于 SegFormer 架构，采用 Mix Transformer (MiT) 作为编码器，具有高效、准确、易于部署的特点。

核心特性

🎯

像素级精确分割

支持多类别语义分割，每个像素都有明确的类别归属，精确定位缺陷区域。

🏛️

6种SegFormer模型

从轻量级B0（3.7M参数）到高精度B5（82M参数），满足不同场景需求。

📁

LabelMe格式支持

原生支持LabelMe标注工具，训练数据和推理结果均为JSON格式，便于查看和编辑。

🪟

滑动窗口推理

支持大图切片推理，自动拼接结果，解决显存限制问题，适用于高分辨率工业图像。

⚡

多格式模型转换

支持导出为TensorRT、ONNX、OpenVINO格式，适配GPU/CPU/边缘设备部署。

📊

完整训练监控

实时显示Loss曲线、mIoU指标、各类别IoU，支持早停和自动保存最优模型。

快速上手

只需5个步骤，即可完成从数据标注到模型训练的完整流程。以下是一个工业缺陷分割的示例。

1
使用 LabelMe 标注数据
打开 LabelMe 工具，使用多边形工具勾画缺陷区域。标签必须使用数字（0, 1, 2...）， 代表不同的缺陷类别。保存后会生成与图像同名的 JSON 文件。
2
组织数据目录
创建两个文件夹：images/ 存放原始图像，jsons/ 存放 JSON 标注文件。 文件名需一一对应（如 001.jpg 对应 001.json）。
3
配置训练参数
在界面中设置类别数（不含背景）、选择模型规格（推荐 B2）、 设置训练轮数（100-300）、批次大小（根据显存调整）。
4
开始训练
点击"开始训练"，系统会自动转换数据格式、下载预训练权重、启动训练。 实时监控 Loss 和 mIoU 指标，训练结束后自动保存最优模型。
5
批量推理
切换到"批量推理"选项卡，选择训练好的模型，指定待预测图像文件夹， 点击开始推理。输出的 LabelMe JSON 文件可直接用于复核或二次标注。
💡
快速验证：建议先用 20-50 张标注图像进行快速训练（20-30 epochs）， 验证标注质量和参数设置是否合理，再进行完整训练。

LabelMe 格式说明

数据目录结构

训练数据需要按以下结构组织：

📁 project/
├── 📁 images/ # 原始图像文件夹
│ ├── 001.jpg
│ ├── 002.jpg
│ ├── 003.png # 支持多种格式
│ └── 004.jpg # 可以没有对应JSON（良品/负样本）
└── 📁 jsons/ # LabelMe JSON文件夹
    ├── 001.json
    ├── 002.json
    └── 003.json

JSON 文件结构

LabelMe 生成的 JSON 文件包含图像信息和多边形标注：

JSON 
{
  "version": "5.2.1",
  "flags": {},
  "shapes": [
    {
      "label": "0",           // 必须是数字字符串！表示类别ID
      "points": [             // 多边形顶点坐标
        [120.5, 80.2],
        [180.3, 75.8],
        [200.1, 150.6],
        [150.0, 180.4],
        [100.2, 120.5]
      ],
      "group_id": null,
      "shape_type": "polygon", // 必须是 polygon
      "flags": {}
    },
    {
      "label": "1",           // 另一个类别
      "points": [...],
      "shape_type": "polygon",
      "flags": {}
    }
  ],
  "imagePath": "001.jpg",
  "imageData": null,          // 建议设为null减小文件体积
  "imageHeight": 1080,
  "imageWidth": 1920
}
🚨
重要：标签必须是数字！
LabelMe 默认使用文字标签，但本模块只支持数字标签（0, 1, 2...）。 请在 LabelMe 中直接输入数字作为标签名，或使用脚本批量转换。

类别ID规则

  • 背景：像素值 255，不需要标注，自动处理
  • 类别ID：从 0 开始连续编号（0, 1, 2...）
  • 类别数：不含背景，只计算实际标注的类别数量
💡
示例：如果你有"划痕"、"凹坑"、"脏污"三种缺陷， 则标签应设为 0、1、2，类别数设为 3。

负样本（良品）

没有缺陷的图像（良品）可以作为负样本参与训练，只需将图像放入 images/ 文件夹， 不创建对应的 JSON 文件即可。系统会自动识别并生成全背景的 Mask。

✏️

标注技巧

标注质量标准

✅

边界精确

多边形顶点应紧贴缺陷边缘，不要留太多空白，也不要包含正常区域。

✅

完整覆盖

确保缺陷区域被完全覆盖，不要遗漏任何部分，尤其是边缘细节。

✅

类别一致

同类型缺陷使用相同的标签ID，不同类型使用不同ID，保持标准统一。

❌

避免重叠

同一图像中的多边形尽量不要重叠，重叠区域会被后绘制的类别覆盖。

数据集规模建议

任务复杂度 类别数 每类样本数 总样本数
简单任务（单一缺陷类型） 1-2 100+ 200+
中等任务（多种缺陷类型） 3-5 200+ 800+
复杂任务（细粒度分类） 5-10 300+ 2000+

数据增强策略

当标注数据有限时，可以通过以下方式扩充数据：

  • 训练时增强：开启水平/垂直翻转、随机旋转、颜色抖动等选项
  • 离线增强：使用 Albumentations 等工具预处理数据
  • 负样本补充：添加足够的良品图像，建议负样本占比 20-30%

转换后的 Mask 是单通道 PNG 图像，每个像素的值代表其类别ID：

模型库

语义分割模块提供 6 种不同规格的 SegFormer 模型，均基于 ImageNet-1K 预训练权重。 SegFormer 采用 Mix Transformer (MiT) 作为编码器，具有高效的多尺度特征提取能力。

SegFormer B0
Backbone: MiT-B0
📦 参数量: 3.7M
⚡ 速度: 最快
最轻量级，适合边缘设备部署，资源受限场景首选。
SegFormer B1
Backbone: MiT-B1
📦 参数量: 13.7M
⚡ 速度: 很快
轻量级，平衡速度与精度，适合实时应用场景。
SegFormer B2
Backbone: MiT-B2
📦 参数量: 24.7M
⚡ 速度: 快
性价比最高，工业首选！在精度和速度之间取得最佳平衡。
SegFormer B3
Backbone: MiT-B3
📦 参数量: 44.6M
⚡ 速度: 中等
高精度，适合复杂场景和细粒度分割任务。
SegFormer B4
Backbone: MiT-B4
📦 参数量: 61.4M
⚡ 速度: 较慢
更高精度，需要较大显存，适合对精度要求极高的场景。
SegFormer B5
Backbone: MiT-B5
📦 参数量: 82.0M
⚡ 速度: 最慢
最高精度，需要大显存（24GB+），追求极致效果。

训练参数

基础参数

参数名 默认值 范围 说明
epochs 100 10-500 训练轮数。小数据集可设100-200，大数据集可设300+
batch_size 8 1-32 批次大小。受显存限制，RTX 3080推荐8-16

val_split 0.2 0.1-0.3 验证集比例。建议0.2（20%用于验证）
patience 20 10-50 早停耐心值。连续N个epoch无改善则停止
save_period 10 5-50 模型保存周期。每N个epoch保存一次

高级参数

参数名 默认值 范围 说明
weight_decay 0.01 0-0.1 权重衰减（L2正则化），防止过拟合

损失函数

本模块采用 CrossEntropy Loss + Dice Loss 组合损失函数， 兼顾像素级分类准确性和区域形状相似度。

损失函数参数

参数名 默认值 说明
loss_ce_weight 0.4 CrossEntropy损失权重。擅长像素级分类
loss_dice_weight 0.6 Dice损失权重。擅长处理类别不平衡和形状匹配
use_class_weight True 是否使用类别权重。自动计算各类别权重缓解不平衡

损失函数说明

📐

CrossEntropy Loss

标准的像素级分类损失，计算每个像素的分类损失并求平均。 对边界细节敏感，是语义分割的基础损失函数。

🎯

Dice Loss

基于Dice系数的损失函数，衡量预测区域与真实区域的重叠度。 对类别不平衡不敏感，适合小目标分割。

💡
调参建议： 默认参数 (CE:0.4, Dice:0.6) 适合大多数场景。 如果边界不够精细，可增加 CE 权重；如果小目标漏检，可增加 Dice 权重。
🎲

数据增强是提升模型泛化能力的重要手段。本模块支持多种在线数据增强方式。

常见问题排查

Loss不下降或震荡严重
可能原因：学习率过大。
解决方案：降低学习率到 1e-5 或 3e-5，或增加 warmup_iters。
mIoU提升后又下降（过拟合）
可能原因：训练轮数过多或数据量不足。
解决方案：减少epochs、增加weight_decay、开启更多数据增强、添加更多训练数据。
某类别IoU始终为0
可能原因：该类别样本过少或标注有误。
解决方案：检查标注是否正确、增加该类别样本数量、开启use_class_weight。
CUDA内存不足
解决方案：减小batch_size、减小img_size、使用更小的模型规格、确保use_fp16开启。

评估指标

语义分割任务使用以下核心指标评估模型性能：

📐
mIoU (Mean Intersection over Union)
所有类别IoU的平均值，是语义分割最核心的指标。计算每个类别预测区域与真实区域的交并比，然后取平均。 范围0-1，越高越好，通常0.7以上可用于生产。
🎯
mDice (Mean Dice Coefficient)
所有类别Dice系数的平均值。Dice系数衡量预测与真实的重叠程度，对小目标更敏感。 范围0-1，通常比mIoU略高。
✅
mAcc (Mean Accuracy)
所有类别像素准确率的平均值。计算每个类别中正确预测的像素比例。 范围0-1，容易受类别不平衡影响。
📉
Loss
训练损失值，由CrossEntropy Loss和Dice Loss加权求和。 越低越好，但需关注验证集指标以防过拟合。

IoU计算公式

公式说明 
IoU = TP / (TP + FP + FN)

其中：
- TP (True Positive): 正确预测为该类别的像素数
- FP (False Positive): 错误预测为该类别的像素数
- FN (False Negative): 漏检的该类别像素数

mIoU = (1/N) × Σ IoU_i  (N为类别数)

ONNX Runtime

ONNX Runtime 是微软开源的跨平台推理引擎，支持多种硬件后端。

推理代码示例

Python 
import numpy as np
import onnxruntime as ort
from PIL import Image

# 加载模型
session = ort.InferenceSession("model.onnx", providers=['CUDAExecutionProvider'])

# 预处理
image = Image.open("test.jpg").convert("RGB")
image = image.resize((512, 512))
input_data = np.array(image).astype(np.float32)

# ImageNet归一化

# 推理
outputs = session.run(None, {"input": input_data})
mask = outputs[0].argmax(axis=1)[0]  # [H, W]

自定义推理流程

Python 
import torch
import numpy as np
from PIL import Image
from mmseg.apis import init_model

# 加载模型
model = init_model('config.py', 'best_model.pth', device='cuda:0')
model.eval()

# 预处理函数
def preprocess(image_path, size=512):
    img = Image.open(image_path).convert('RGB')
    img = img.resize((size, size))
    img_array = np.array(img).astype(np.float32)
    
    # ImageNet归一化
    mean = np.array([123.675, 116.28, 103.53])
    std = np.array([58.395, 57.12, 57.375])
    img_array = (img_array - mean) / std
    
    # HWC -> CHW -> NCHW
    img_tensor = torch.from_numpy(img_array.transpose(2, 0, 1))
    img_tensor = img_tensor.unsqueeze(0).float().cuda()
    return img_tensor

# 推理
with torch.no_grad():
    input_tensor = preprocess('test.jpg')
    output = model.encode_decode(input_tensor, None)
    mask = output.argmax(dim=1).cpu().numpy()[0]

# mask 是 [H, W] 的numpy数组，值为类别ID

常见问题

标签必须是数字吗？能用文字标签吗？
是的，必须使用数字标签（0, 1, 2...）。如果已有文字标签的数据， 需要先编写脚本将标签转换为数字。建议维护一个类别映射表（如：划痕=0, 凹坑=1）。
mIoU一直是0怎么办？
可能原因：
1. 类别数设置错误：检查设置的类别数是否与实际标注的类别一致
2. 标签ID超出范围：如果设置类别数为3，但标注中有标签"5"，则会被忽略
3. 标签不是数字：检查JSON中的label字段是否为数字字符串
4. 数据路径错误：确认图像和JSON文件能正确配对
类别不平衡（某类样本很少）怎么处理？
1. 开启 use_class_weight 选项，自动计算类别权重
2. 增加 Dice Loss 权重（如 CE:0.3, Dice:0.7）
3. 对少数类别进行过采样或数据增强
4. 收集更多该类别的样本
TensorRT转换失败怎么办？
1. 检查TensorRT版本是否与CUDA版本匹配
2. 尝试降低workspace_gb参数
3. 先转换为ONNX格式，再用trtexec工具转换
4. 检查模型中是否有不支持的算子
小目标检测效果差怎么办？
1. 增大 img_size（如 768 或 1024）
2. 使用滑动窗口推理保留原始分辨率
3. 增加 Dice Loss 权重
4. 确保标注质量，小目标边界要精确
5. 降低 min_area 阈值以保留小区域
